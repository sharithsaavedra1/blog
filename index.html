Etiquetas Estructurales

Dan forma y semántica al esqueleto de una página web.

<html> Raíz

Definición

Es el elemento raíz de todo documento HTML. Envuelve todo el contenido.

Sintaxis

<html lang="es">
  ...contenido...
</html>

Buenas prácticas

  • Siempre incluir el atributo lang para accesibilidad.
  • Debe contener solo <head> y <body>.
<head> Metadatos

Definición

Contiene los metadatos: título, estilos y scripts. No se muestra en pantalla.

Sintaxis

<head>
  <meta charset="UTF-8">
  <title>Mi página</title>
  <link rel="stylesheet" href="css.css">
</head>

Buenas prácticas

  • Incluir siempre charset, viewport y title.
  • Cargar CSS antes de cerrar el <head>.
<body> Contenido

Definición

Contiene todo el contenido visible de la página.

Sintaxis

<body>
  <h1>Hola mundo</h1>
  <p>Bienvenido al blog.</p>
</body>

Buenas prácticas

  • Solo puede haber un <body> por documento.
  • Cargar scripts antes de cerrar </body>.
<header> <main> <footer> Semántica

Definición

Etiquetas semánticas que delimitan las zonas principales del sitio.

Sintaxis

<header> Mi Blog</header>
<main>
  <p>Contenido principal</p>
</main>
<footer>© 2025</footer>

Demo visual

HEADER 🌸
MAIN — contenido
FOOTER

Buenas prácticas

  • Mejoran accesibilidad y SEO.
  • Reemplaza <div id="header">.

Etiquetas de Texto

Formatean y estructuran el contenido textual del documento.

<h1> — <h6> Encabezados

Definición

Definen títulos y subtítulos. h1 es el más importante; h6 el menos.

Sintaxis

<h1>Título principal</h1>
<h2>Subtítulo</h2>
<h3>Sección</h3>

Demo visual

H1 — Título grande

H2 — Subtítulo

H3 — Sección pequeña

Buenas prácticas

  • Solo un <h1> por página para SEO.
  • No saltar niveles de jerarquía.
<p> Párrafo

Definición

Define un párrafo de texto con márgenes automáticos.

Sintaxis

<p>Este es un párrafo de ejemplo.</p>

Buenas prácticas

  • No anidar párrafos dentro de otros párrafos.
  • Usar para bloques de texto continuo.
<strong> & <em> Énfasis

Definición

<strong> indica importancia (negrita). <em> indica énfasis (cursiva).

Sintaxis

<p>Es <strong>muy importante</strong>
estudiar <em>todos los días</em>.</p>

Demo visual

Es muy importante estudiar todos los días.

Buenas prácticas

  • Preferir strong/em sobre b/i por semántica.
<a> Enlace

Definición

Crea hipervínculos a otras páginas, archivos o secciones.

Sintaxis

<a href="https://developer.mozilla.org"
   target="_blank">
  Visita MDN 
</a>

Demo visual

Visita MDN

Buenas prácticas

  • Usar texto descriptivo, no "haz clic aquí".
  • Agregar rel="noopener" con target="_blank".

Etiquetas de Listas

Organizan información en grupos ordenados o no ordenados.

<ul> + <li> No ordenada

Definición

<ul> crea una lista con viñetas. Cada elemento va en un <li>.

Sintaxis

<ul>
  <li>HTML</li>
  <li>CSS</li>
  <li>JavaScript</li>
</ul>

Demo visual

  • HTML
  • CSS
  • JavaScript

Buenas prácticas

  • Usar cuando el orden no importa.
  • Los <li> solo van dentro de ul/ol.
<ol> + <li> Ordenada

Definición

<ol> crea una lista numerada donde el orden tiene significado.

Sintaxis

<ol>
  <li>Abre VS Code</li>
  <li>Crea index.html</li>
  <li>Escribe tu código</li>
</ol>

Demo visual

  1. Abre VS Code
  2. Crea index.html
  3. Escribe tu código

Buenas prácticas

  • Usar para pasos o instrucciones.
  • Atributo start para empezar en número diferente.
<dl> <dt> <dd> Definición

Definición

Lista de definiciones: dt es el término y dd su descripción.

Sintaxis

<dl>
  <dt>HTML</dt>
  <dd>Lenguaje de marcado web</dd>
  <dt>CSS</dt>
  <dd>Lenguaje de estilos</dd>
</dl>

Demo visual

HTML
Lenguaje de marcado web
CSS
Lenguaje de estilos

Buenas prácticas

  • Usar para glosarios o pares clave-valor.
  • No usar solo para sangría visual.

Etiquetas de Formularios

Recopilan datos del usuario para enviarlos al servidor.

<form> Contenedor

Definición

Envuelve todos los controles de un formulario y define cómo se envían los datos.

Sintaxis

<form action="/enviar" method="post">
  ...controles...
</form>

Buenas prácticas

  • Siempre especificar method (get/post).
  • Usar novalidate solo con validación propia.
<input> Campo

Definición

Control más versátil de HTML. Su comportamiento cambia con el atributo type.

Sintaxis

<input type="text" placeholder="Tu nombre">
<input type="email" required>
<input type="password">
<input type="checkbox">
<input type="radio" name="g1">

Demo visual

Buenas prácticas

  • Asociar siempre un <label> con cada input.
  • Usar el type correcto para mejor UX móvil.
<select> + <option> Desplegable

Definición

Crea un menú desplegable con opciones predefinidas.

Sintaxis

<select name="nivel">
  <option value="basico">Básico</option>
  <option value="intermedio">Intermedio</option>
  <option value="avanzado">Avanzado</option>
</select>

Demo visual

<textarea> Área de texto

Definición

Campo de texto multilínea para mensajes largos o comentarios.

Sintaxis

<textarea rows="4"
  placeholder="Escribe aquí...">
</textarea>

Buenas prácticas

  • Definir tamaño con CSS en lugar de rows/cols.
  • Añadir maxlength para limitar caracteres.

Etiquetas de Tablas

Organizan datos en filas y columnas de forma estructurada.

<table> <tr> <th> <td> Tabla completa

Definición

table crea la tabla. tr = fila, th = encabezado, td = celda de datos.

Sintaxis

<table>
  <thead>
    <tr>
      <th>Etiqueta</th>
      <th>Uso</th>
    </tr>
  </thead>
  <tbody>
    <tr>
      <td><table></td>
      <td>Contenedor de tabla</td>
    </tr>
  </tbody>
</table>

Demo visual

EtiquetaFunción
<table>Contenedor principal
<tr>Fila
<th>Celda encabezado
<td>Celda dato

Buenas prácticas

  • Usar thead, tbody y tfoot para semántica.
  • Las tablas son para DATOS, no para maquetar.
  • Incluir scope en los th para accesibilidad.

Etiquetas Multimedia

Insertan imágenes, videos, audio y otros recursos visuales.

<img> Imagen

Definición

Inserta una imagen. Es una etiqueta vacía, no tiene cierre.

Sintaxis

<img src="foto.jpg"
     alt="Descripción"
     width="300"
     height="200">

Demo visual

Buenas prácticas

  • El atributo alt es OBLIGATORIO para accesibilidad.
  • Especificar width y height para evitar saltos.
  • Usar formatos modernos: WebP, AVIF.
<video> Video

Definición

Inserta un reproductor de video sin plugins externos.

Sintaxis

<video src="video.mp4"
       controls
       width="400"
       poster="miniatura.jpg">
  Tu navegador no soporta video.
</video>

Demo visual

Buenas prácticas

  • Siempre incluir controls para el usuario.
  • Añadir subtítulos con <track>.
  • Usar <source> para múltiples formatos.
<audio>

Definición

Inserta un reproductor de audio. Soporta MP3, OGG y WAV.

Sintaxis

<audio controls>
            <source src="cancion.mp3" type="audio/mpeg">
            <source src="cancion.ogg" type="audio/ogg">
            </audio>

Buenas prácticas

  • Proveer mínimo 2 formatos para compatibilidad.
  • No usar autoplay sin permiso del usuario.
<iframe> Incrustado

Definición

Incrusta otra página web, mapa o video de YouTube dentro de la página.

Sintaxis

<iframe
            src="https://youtu.be/1bcoeAALVgE?si=00gure2Ky8JAhTNs"
             width="560" height="315"
             allowfullscreen
             title="Mi video">
                </iframe>

Buenas prácticas

  • Siempre incluir atributo title.
  • Usar sandbox para limitar permisos.
  • Carga diferida con loading="lazy".
